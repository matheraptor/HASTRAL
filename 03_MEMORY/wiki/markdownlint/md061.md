# md061

**Summary**: Top-level heading requirement (MD041) is deferred when YAML frontmatter is present.

**Sources**: (source: Project Sovereign Doctrine)

**Last updated**: 2026-05-16

---

## Rule Definition

When a Markdown file begins with a YAML frontmatter block (delimited by `---`), the requirement for the first line of the file to be a top-level heading (MD041) is modified.

### Correct Structure

1. Opening `---` delimiter.
2. YAML metadata.
3. Closing `---` delimiter.
4. **Top-level heading (`# Heading`)** must immediately follow the closing delimiter.

### Violation

- **CRITICAL**: Placing the top-level heading *above* the opening `---` delimiter. This breaks the YAML parser and renders the frontmatter invalid.
- Omitting the top-level heading entirely after the frontmatter block.

## Rationale

YAML frontmatter is a structural requirement for metadata processing in the HASTRAL/MAGPIE ecosystem. Moving the heading above the frontmatter would break the parser's ability to identify the metadata block.
