# Operational Hierarchy

**Summary**: The structured framework for mission planning and execution, ensuring a single purpose at every level of activity to prevent scope creep.

**Sources**: (source: OPERATIONAL_HIERARCHY.md)

**Last updated**: 2026-04-15

---

## 1. Hierarchy Structure

The HASTRAL operational framework defines five distinct levels of activity, each mapped to a specific purpose type.

| LEVEL         | PURPOSE   | DEFINITION                                      |
| :------------ | :-------- | :---------------------------------------------- |
| **STEP**      | FUNCTION  | Atomic action with a single executable function |
| **TASK**      | INTENT    | Sequence of steps achieving a single intent     |
| **MISSION**   | OBJECTIVE | Collection of tasks achieving an objective      |
| **OPERATION** | GOAL      | Collection of missions achieving a goal         |
| **CAMPAIGN**  | DOCTRINE  | Collection of operations aligned to doctrine    |

## 2. The Single Purpose Rule

**Core Principle**: Each level has **ONE and ONLY ONE** purpose of its type.

- **STEP** $\rightarrow$ ONE FUNCTION
- **TASK** $\rightarrow$ ONE INTENT
- **MISSION** $\rightarrow$ ONE OBJECTIVE
- **OPERATION** $\rightarrow$ ONE GOAL
- **CAMPAIGN** $\rightarrow$ ONE DOCTRINE

**Rationale**: This strict separation prevents scope creep and maintains absolute clarity. If an activity requires multiple purposes at the same level, it _must_ be fragmented into separate entities.

## 3. Fragmentation Logic

- **Single Task**: A TASK (Intent: X) is decomposed into multiple STEPs (Functions: A, B, C, D).
- **Complex Mission**: A MISSION (Objective: Z) is decomposed into multiple TASKs (Intents: X, Y), each of which contains its own sequence of STEPs.
