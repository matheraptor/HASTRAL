# Urgency and Gravity Scales

**Summary**: The standardized metrics for prioritizing activities and assessing the impact of issues within the HASTRAL ecosystem.

**Sources**: (source: URGENCY.md, GRAVITY.md)

**Last updated**: 2026-05-19

---

## Urgency Scale

Urgency defines the temporal requirement for action (When must this be done?).

| LEVEL | LABEL           | DESCRIPTION                                         |
| :---- | :-------------- | :-------------------------------------------------- |
| 0     | **STRATEGIC**   | Reference for long-term alignment; neutral urgency. |
| 1     | **IMMEDIATE**   | Must do **NOW**.                                    |
| 2     | **DIRE**        | Must do **NEXT**.                                   |
| 3     | **CRITICAL**    | Must do **AS SOON AS POSSIBLE (ASAP)**.             |
| 4     | **SIGNIFICANT** | Should do **SOON**.                                 |
| 5     | **LATENT**      | Can do if and when possible.                        |

## Gravity Scale

Gravity defines the impact or weight of an issue/activity (How important is this?).

| LEVEL | LABEL         | RELATIONSHIP                                           |
| :---- | :------------ | :----------------------------------------------------- |
| 0     | **TACTICAL**  | Neutral gravity; baseline context.                     |
| 1     | **VITAL**     | Must care about this ABOVE ALL.                        |
| 2     | **SEVERE**    | Must care about this AS MUCH AS POSSIBLE.              |
| 3     | **PRESSING**  | Should care about this when possible.                  |
| 4     | **IMPORTANT** | Might care sometimes; probably unsafe to ignore.       |
| 5     | **TRIVIAL**   | "Probably safe to ignore".                             |
