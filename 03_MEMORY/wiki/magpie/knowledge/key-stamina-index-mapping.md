# Key Stamina Index Mapping

**Summary**: Mapping definition between `keyID` and `staminaIndex` for efficient `handIndex` resolution.

**Sources**: 
- [[magpie-key-system]]

**Last updated**: 2026-05-18

---

The system maps `MAGPIE_KEY` identifiers of type `AXIOM` to specific `staminaIndex` locations within the entity's `fitness` array. This allows for constant-time resolution of trait or card context during event processing.

## Resolution Pattern

When an `EXP` is generated by a player-initiated card action, the corresponding `staminaIndex` (0-9) is mapped to a pre-defined `AXIOM` key ID. 

### Mapping

| Key Label | `keyID` (Axiom) | `staminaIndex` |
| :--- | :--- | :--- |
| `HAND_INDEX_0` | `...` | 0 |
| `HAND_INDEX_1` | `...` | 1 |
| ... | ... | ... |
| `HAND_INDEX_9` | `...` | 9 |

The resolution is performed using the `MAGPIE.KEY.INDEX.STAMINA` Map:

```javascript
const handIndex = exp.keys.find(keyID => MAGPIE.KEY.INDEX.STAMINA.get(keyID))
```
